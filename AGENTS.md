# Coding Agent 工作说明

## 项目目标

实现 Skyrim Mod Workbench 第一版：一个本地优先的 Windows 桌面工具，用于将多个真实文件组装为逻辑 Mod 条目，并通过预览、中文信息、目录树和安全移动完成分类归档。

## 不可违反的规则

1. 不得上传、同步或发送用户 Mod 文件、路径、图片或数据库内容。
2. 不得实现未经确认的自动文件移动、删除、覆盖或重命名。
3. 不得让 UI 直接调用 shutil、os.rename、Path.rename 或其他文件写操作。
4. 所有文件移动必须先生成并保存预演计划。
5. 所有文件移动必须支持安全撤销预演。
6. 不得假设文件名有统一格式。
7. 不得自动把本体、汉化、预览图合并为同一 Mod 条目；只能提供候选或人工操作。
8. 不得修改用户原始图片、压缩包、文档或 Mod 内容。
9. 不得把缩略图缓存写入用户 Mod 目录。
10. 所有新功能必须优先支持中文路径和 UTF-8 JSON。
11. 不得扩展到云端、账号、MO2 管理、自动爬取 Nexus、压缩包内容解析或未在规格中定义的功能。
12. 所有待确认需求必须保留 TODO 或明确注释，不得自行假定产品决策。

## 开发方式

* 每个原子任务完成后运行测试。
* 每次改动保持小而可审查。
* 优先编写领域逻辑与测试，再接入 UI。
* 对涉及真实文件的测试，必须使用 pytest 临时目录。
* 不得用真实用户目录作为测试目录。
* 所有异常必须转换为用户可理解的错误信息，并保留技术日志。

## 代码质量

* 使用类型标注。
* 使用 pathlib.Path。
* 使用 ruff 格式化和静态检查。
* 核心文件操作必须有单元测试。
* 数据库 schema 变更必须使用迁移或可重复初始化逻辑。
* UI 文本优先使用中文；待确认是否需要英文国际化。

## 完成定义

一个功能只有在以下条件都满足时才算完成：

* 有明确输入、输出和失败行为。
* 有至少一个自动化测试，文件操作除外的纯 UI 微调可例外。
* 不会绕过文件操作预演。
* 不会破坏中文路径。
* 不会引入未讨论的产品范围。
* 文档或注释说明了关键约束。
