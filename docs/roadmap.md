## 阶段 2：新骨架 + 基础浏览 ⏳

目标：建立新数据库模型，实现增量扫描和基础目录浏览。

### Task 1：新数据库 Schema + 迁移 ✅

> 完成于 2026-07-12，版本 v0.10.0。schema_version 由 3 升至 4。

- 创建新表：`content_unit`、`tag_category`、`tag`、`content_unit_tag`、`operation_history`、`folder_cache`
- 保留表：`managed_root`、`thumbnail_cache`（FK 由 `asset_id`→`file_asset` 改为 `content_unit_id`→`content_unit`）
- 移除旧表：`mod_item`、`file_asset`、`folder_node`、`operation_log`
- v3→v4 迁移函数，幂等
- 数据库位置：`%LOCALAPPDATA%\SkyrimContentWorkbench\app.db`（旧名 `SkyrimModWorkbench` 已废弃）
- 应用数据目录改名：`SkyrimModWorkbench` → `SkyrimContentWorkbench`

**验收：**
- [x] v3→v4 迁移创建全部新表且幂等
- [x] 旧表（mod_item、file_asset、folder_node、operation_log）已被移除
- [x] 保留表（managed_root）数据不受影响；thumbnail_cache 重建（FK 改为 content_unit_id）
- [x] 所有新表支持 Unicode 和中文路径
- [x] 迁移函数可重复执行（幂等）

### Task 2：新扫描器 ✅

> 完成于 2026-07-12，版本 v0.11.0。

**2026-07-13 设计修正**：扫描规则改为"所有压缩包文件自动标记为内容单元，文件夹不自动标记"。

- 保留 FileScanner 的递归扫描能力
- 增量扫描：启动时对比目录 mtime，跳过未变更目录
- 内容单元识别规则：所有压缩包文件（.7z/.zip/.rar 等）自动标记为内容单元
- 写入 FolderCache（目录树性能缓存）
- 候选内容单元写入 content_unit 表（status=unorganized）
- 跳过已被标记为内容单元的目录内部

**验收：**
- [x] 增量扫描基于 mtime 正确跳过未变更目录
- [x] 含压缩包的文件夹被识别为内容单元候选
- [x] 不含压缩包的分类目录不被错误识别

### Task 5：快速插入 ✅

- 整理模式下选中目标目录 → 点击「快速插入」按钮 → 当前装配面板绑定的 Mod 组整体移入目标目录
- 移动安全规则（spec §6.1）：确认弹窗、冲突处理（当前阶段拒绝 + 提示，覆盖/跳过/重命名留待阶段 5）、跨盘提示、子目录阻止
- 操作记录写入 operation_history
- ContentUnit.path 同步更新；folder_cache 旧路径记录清理

**设计决策（2026-07-17 用户确认）：**
- **取消拖拽方案**：原 roadmap 中"拖拽到目录树节点触发文件移动"和"目录树内拖拽重新分类"取消，与 Task 4 拖拽取消决策一致（Qt 拖拽反馈不理想）。仅保留「快速插入」按钮入口。
- **冲突处理范围**：Task 5 保持现有"冲突即拒绝"模式（ConflictError + 弹窗提示），覆盖/跳过/重命名选项留待阶段 5 通用冲突处理统一实现（AGENTS 规则 2：不覆盖）。
- **跨盘移动**：当前阶段直接拒绝并提示，不执行跨盘 move（Windows 跨盘 move 退化为 copy+delete，风险较高）。

**验收：**

### Task 2：排序

- 排序字段：名称 / 修改日期 / 类型 / 大小 / 整理状态
- 正序/倒序切换

**验收：**
- [ ] 各排序字段正常工作
- [ ] 正序/倒序切换正确
- [ ] 中文名称排序合理

### Task 3：重命名 + 删除

- 右键重命名（真实文件夹改名 + 同步更新 content_unit.path）

- 应用外重命名后的路径丢失检测与手动重新关联

**验收：**
- [ ] 重命名同时修改真实文件夹名和数据库中的 path
- [ ] 删除操作将文件移至回收站
- [ ] 应用外重命名后扫描可检测路径丢失

### Task 4：操作历史与撤销

- operation_history 正确记录所有操作

**验收：**

- [ ] 可从历史记录中选择操作撤销

- [ ] 撤销后文件回到操作前位置

### Task 5：全局搜索

- 搜索范围：内容单元标题 + 标签名称 + 备注

- 搜索结果列表，点击跳转到对应内容单元

**验收：**
- [ ] 搜索标题/标签/备注命中正确结果
- [ ] 中文搜索正常
- [ ] 点击搜索结果跳转到内容单元所在目录

## 版本约定

在 1.0.0 之前，0.MINOR.PATCH 中的 MINOR 用于标记阶段推进，PATCH 用于同阶段内的修复与小幅调整。

| 版本 | 对应 |
|------|------|
| v0.1.0 | 阶段 0 ✅ |
| v0.2.0~v0.5.0 | 阶段 1 ✅ |
| v1.0.0 | 阶段 6 完成后 |

旧版 v0.6.0~v0.9.0（阶段 2 旧架构）已不再使用。
