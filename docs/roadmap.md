## 阶段 1：安全数据与文件操作基础 ✅

> 完成于 2026-07-07，版本 v0.2.0~v0.5.0。

目标：建立扫描、数据模型和安全移动能力，不先做复杂 UI。

范围：
- [x] 数据库 Schema v1（4 表 + Repository CRUD）
- [x] 领域模型（ModItem、FileAsset、FolderNode、OperationLog）
- [x] 只读文件扫描器（递归扫描 + 扩展名分类 + 中文路径）
- [x] Mod 条目组装服务（创建 ModItem、关联/移除成员、设角色、设封面）
- [x] 安全文件操作服务（移动预演 → 确认 → 执行 → 撤销）
- [x] 操作日志与撤销预演（B1/B2 安全校验）
- [x] 中文路径测试

> **注意：** 阶段 0 和阶段 1 的旧版代码（ModItem/FileAsset 体系）在新架构下需要逐步改造或重写。阶段 2 起所有新开发按新架构进行。

## 阶段 2：新骨架 + 基础浏览 ⏳

目标：建立新数据库模型，实现增量扫描和基础目录浏览。

### Task 1：新数据库 Schema + 迁移 ✅

> 完成于 2026-07-12，版本 v0.10.0。schema_version 由 3 升至 4。

- 创建新表：`content_unit`、`tag_category`、`tag`、`content_unit_tag`、`operation_history`、`folder_cache`
- 保留表：`managed_root`、`thumbnail_cache`（FK 由 `asset_id`→`file_asset` 改为 `content_unit_id`→`content_unit`）
- 移除旧表：`mod_item`、`file_asset`、`folder_node`、`operation_log`
- v3→v4 迁移函数，幂等
- 数据库位置：`%LOCALAPPDATA%\SkyrimContentWorkbench\app.db`（旧名 `SkyrimModWorkbench` 已废弃）
- 应用数据目录改名：`SkyrimModWorkbench` → `SkyrimContentWorkbench`

**验收：**
- [x] v3→v4 迁移创建全部新表且幂等
- [x] 旧表（mod_item、file_asset、folder_node、operation_log）已被移除
- [x] 保留表（managed_root）数据不受影响；thumbnail_cache 重建（FK 改为 content_unit_id）
- [x] 所有新表支持 Unicode 和中文路径
- [x] 迁移函数可重复执行（幂等）

### Task 2：新扫描器

- 保留 FileScanner 的递归扫描能力
- 增量扫描：启动时对比目录 mtime，跳过未变更目录
- 内容单元识别规则：含压缩包文件（.7z/.zip/.rar 等）的文件夹 → 候选内容单元
- 写入 FolderCache（目录树性能缓存）
- 候选内容单元写入 content_unit 表（status=unorganized）
- 跳过已被标记为内容单元的目录内部

**验收：**
- [ ] 增量扫描基于 mtime 正确跳过未变更目录
- [ ] 含压缩包的文件夹被识别为内容单元候选
- [ ] 不含压缩包的分类目录不被错误识别
- [ ] 扫描结果持久化到 content_unit 和 folder_cache
- [ ] 支持中文目录名和文件名
- [ ] 可手动触发全量重扫

### Task 3：目录树浏览

- 从 FolderCache 读取目录树结构
- 保留 FolderTreeModel 惰性加载架构（canFetchMore / fetchMore）
- 适配新数据源（folder_cache 表替代旧 folder_node）
- 目录树只展示基础分类（约一级或两级）
- 支持中文目录名

**验收：**
- [ ] 目录树从 FolderCache 正确加载
- [ ] 惰性加载正常（展开节点时查询子节点）
- [ ] 中文目录名正确显示
- [ ] 目录树不崩溃（错误节点降级为空子树）
- [ ] 重启应用后目录树从缓存加载，无需重新扫描

### Task 4：内容单元列表 + 浏览模式

- 选中目录 → 中间显示该目录下的内容单元列表（来源 content_unit 表）
- 列表显示：标题/路径、类型图标、封面缩略图、整理状态
- 基础浏览模式框架
- 双击内容单元 → 右栏显示元数据（空状态）
- 新建右栏元数据面板占位

**验收：**
- [ ] 点击目录树节点 → 中间显示对应的内容单元列表
- [ ] 内容单元正确显示标题、类型图标、状态
- [ ] 双击内容单元 → 右栏显示该单元的元数据（即使未编辑过）
- [ ] 无内容单元的目录显示空状态
- [ ] 中文标题正确显示

### Task 5：双模式切换 + 扫描联动

- 顶部 [浏览|整理] 模式切换按钮
- 整理模式下中间区固定（暂存区展示），目录树变为目标选择器
- 扫描完成 → 目录树刷新 + 新候选内容单元出现

**验收：**
- [ ] 浏览/整理模式切换正常工作
- [ ] 整理模式下点击目录树节点不切换中间内容
- [ ] 整理模式下目录树高亮显示选中的目标路径
- [ ] 扫描完成后目录树自动刷新
- [ ] 新扫描出的内容单元候选自动出现在目录中

## 阶段 3：暂存区 + 整理工作流

目标：实现从暂存区整理文件到分类目录的完整工作流。

### Task 1：暂存区标记与管理

- 右键目录 → "标记为暂存区" / "取消暂存区标记"
- 支持多个暂存区同时存在
- 暂存区在目录树中有视觉标记（如 [S] 图标或特殊颜色）
- 暂存区配置持久化到数据库

**验收：**
- [ ] 右键目录可标记/取消暂存区
- [ ] 多个目录可同时标记为暂存区
- [ ] 暂存区在目录树中可辨识
- [ ] 重启应用后暂存区标记保留

### Task 2：暂存区文件列表

- 选中暂存区 → 中间显示其下所有文件（含子目录中的文件）
- 文件列表显示：文件名、类型图标、大小、修改日期
- 支持排序：名称/日期/类型/大小
- 支持中文文件名

**验收：**
- [ ] 选中暂存区显示其中所有文件
- [ ] 文件信息正确（名称、类型图标、大小、日期）
- [ ] 排序功能正常
- [ ] 中文文件名正确显示

### Task 3：创建 Mod 组

- 右键文件 → "创建 Mod 组"
- 自动从文件名提取主要名称（剔除版本号、后缀、网址标注等无用信息）
- 下拉框提供选项：纯 Mod 名 / 完整原名（参考 MO2 安装 Mod 时的起名逻辑）
- 以所选名字在暂存区创建真实文件夹
- 将源文件移入该文件夹
- 新建内容单元，路径指向新文件夹

**验收：**
- [ ] 右键文件弹出"创建 Mod 组"菜单项
- [ ] 自动从文件名正确提取主要名称（如 "BDOR Black Knight 1.0.7z" → "BDOR Black Knight"）
- [ ] 下拉框提供两种命名选项
- [ ] 真实文件夹在暂存区中被创建
- [ ] 源文件被移入新文件夹
- [ ] 内容单元在数据库中被创建（status=unorganized）
- [ ] 暂存区文件列表刷新，新文件夹出现

### Task 4：装配面板

- 创建 Mod 组后，中间内容区分割出"正在组装"子面板
- 面板显示当前 Mod 组文件夹的内容
- 从上方暂存区文件列表拖入附加文件（汉化、预览图等）
- 支持移除已加入的文件
- 拖入的预览图文件自动重命名为文件夹名

**验收：**
- [ ] 创建 Mod 组后装配面板自动出现
- [ ] 装配面板显示文件夹内已有文件
- [ ] 可从暂存区拖入文件到装配面板
- [ ] 拖入后文件真实移动到 Mod 组文件夹
- [ ] 可移除已加入的文件（文件移回暂存区）
- [ ] 预览图文件拖入后自动重命名

### Task 5：快速插入 + 拖拽移动

- 整理模式下选中目标目录 → "快速插入"按钮 → 当前 Mod 组移入目标
- 从暂存区直接拖拽文件/文件夹到目录树的目标节点
- 目录树内拖拽重新分类
- 移动安全规则：确认弹窗、冲突处理（覆盖/跳过/重命名）、跨盘提示、子目录阻止
- 操作记录写入 operation_history

**验收：**
- [ ] 快速插入按钮选中目标后正常工作
- [ ] 拖拽到目录树节点触发文件移动
- [ ] 目录树内拖拽触发重新分类
- [ ] 移动前弹出确认弹窗
- [ ] 重名冲突时弹窗提供覆盖/跳过/重命名选项
- [ ] 跨盘移动时提示用户
- [ ] 文件夹移到自身或子目录时被阻止
- [ ] 每次操作写入 operation_history

## 阶段 4：元数据 + 标签系统

目标：为内容单元建立完整的元数据标记和标签筛选能力。

### Task 1：标签分类管理

- 标签分类 CRUD（创建、读取、更新、删除）
- 每个分类有名称和色相值（color_hue）
- 预置 Skyrim 常用标签库：
  - 服装护甲：重甲、轻甲、法袍、现代、幻想
  - 武器：单手剑、双手剑、弓、法杖
  - 作者：Kirax、NINI、3HY 等
  - 来源：N 网、群友分享、自购
  - 状态：已测试、已汉化、待测试、推荐
  - 部位：身体、头部、腿部

**验收：**
- [ ] 标签分类可增删改查
- [ ] 颜色设置生效
- [ ] 预置标签库正确加载

### Task 2：打标签

- 给单个内容单元添加/移除标签
- 打字输入 + 自动补全已有标签
- 批量打标签（选中多个内容单元 → 一次性添加/移除标签）
- 支持中文标签

**验收：**
- [ ] 单个内容单元可添加/移除标签
- [ ] 输入时自动补全已有标签
- [ ] 批量选中内容单元后打标签生效于全部
- [ ] 中文标签正常

### Task 3：标签筛选

- 浏览模式下顶部标签筛选栏
- 先选标签分类 → 展开该分类下的标签
- 多选标签 → 内容单元列表实时筛选
- 多分类标签间为 AND 关系

**验收：**
- [ ] 标签筛选栏显示所有分类
- [ ] 点击分类展开对应标签
- [ ] 多选标签后内容单元列表实时更新
- [ ] 不同分类的标签组合筛选正确
- [ ] 无结果的空状态显示

### Task 4：元数据编辑

- 编辑标题（中文显示名）
- 编辑来源 URL、评分、备注
- 设置封面图片（从内容单元目录内选择图片 + 自动候选第一张）
- 整理状态切换（未整理 / 已整理）

**验收：**
- [ ] 标题/来源/评分/备注可编辑保存
- [ ] 封面可从目录中选取
- [ ] 第一张图片自动候选为封面
- [ ] 整理状态切换生效
- [ ] 重启应用后元数据保留

### Task 5：封面预览

- 内容单元列表中以小图标显示封面
- 浏览模式下大图卡片展示封面
- 缩略图缓存（复用 ThumbnailGenerator，关联改为 content_unit_id）

**验收：**
- [ ] 列表中小图标显示封面
- [ ] 大图卡片正确展示
- [ ] 无封面时显示占位
- [ ] 缩略图缓存写入应用目录
- [ ] 源图不被修改

- 排序字段：名称 / 修改日期 / 类型 / 大小 / 整理状态

**验收：**

- [ ] 正序/倒序切换正确

## 版本约定

在 1.0.0 之前，0.MINOR.PATCH 中的 MINOR 用于标记阶段推进，PATCH 用于同阶段内的修复与小幅调整。

| 版本 | 对应 |
|------|------|
| v0.1.0 | 阶段 0 ✅ |
| v0.2.0~v0.5.0 | 阶段 1 ✅ |
| v1.0.0 | 阶段 6 完成后 |

旧版 v0.6.0~v0.9.0（阶段 2 旧架构）已不再使用。
