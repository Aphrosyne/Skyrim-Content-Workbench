# Skyrim Content Workbench — 产品规格说明

> 本文档为方向 C 确认后的重写版。旧版已归档至 `archive/`。
>
> 设计决策来源：`docs/product-review.md`（产品复盘）、`docs/design-workbook.md`（设计工作手册）

## 1. 项目定位

Skyrim Content Workbench 是一个面向 Windows 的、本地优先（local-first）的本地数字资产管理工具（DAM, Digital Asset Management）。第一阶段专门优化 Skyrim Mod 的整理与浏览工作流，但核心模型（内容单元 + 路径 + 元数据）可扩展到教程、截图、视频项目等其他资产类型。

它不替代 Mod Organizer 2、Everything 或 Windows 资源管理器。它解决的核心问题是：用户有大量零散放置的本地数字资产（Mod 文件、教程、素材、截图），需要通过预览、中文信息、标签和可交互的目录树，快速完成识别、标记、分类与归档。

项目计划开源，第一版不包含云同步、账号系统或用户数据上传。

## 2. 核心原则

### 2.1 模型原则

- **真实文件系统是唯一的事实来源（Single Source of Truth）**：文件始终是普通文件，能够脱离本程序继续被资源管理器、MO2 和其他工具使用。
- **数据库是元数据增强层**：数据库不是另一套文件组织体系，仅保存目录/文件自身无法表达的信息（中文显示名、标签、备注、来源 URL、封面关联、搜索索引、AI 分析结果）。
- **内容单元替代 ModItem + FileAsset**：数据库记录的是"内容单元"（一个文件夹或一个单文件），不是每个文件记录，也不是虚拟条目。
- **标签替代深层目录**：目录树保持扁平（建议最多二级分类），细分类由标签系统承担。

### 2.2 安全原则

- 所有文件操作直接作用于真实文件系统（不再是预演→确认→执行的模式），但保留必要的安全机制。
- 文件移动前弹出确认提示；重名冲突时弹窗让用户选择（覆盖/跳过/重命名）。
- 禁止将文件夹移动到自身或子目录。
- 跨盘移动需明确提示用户。
- 删除操作统一移至 Windows 回收站。
- 所有操作记录到简化版操作历史表，支持安全撤销。

### 2.3 中文优先

- 真实路径和文件名可为中文（MO2 对中文路径支持良好，无需回避）。
- 内容单元支持中文别名（title 字段），可选覆盖真实文件夹名。
- 所有 UI 优先使用中文显示。
- 搜索完整支持中文和 UTF-8。

### 2.4 技术原则

- Python 3.12+ / PySide6 / SQLite + FTS5 / Pillow
- local-first：所有数据保存在本机，不上传用户文件、路径、图片或元数据
- 不依赖统一命名规则：不假设 Nexus Mods、汉化包、社区分享文件或预览图存在可靠命名规律
- 不读写压缩包内部内容
- 不修改用户原图（缩略图缓存写入应用数据目录）

## 3. 第一版目标

第一版的目标是让用户能够完成一次完整的"暂存区整理 → 元数据标记 → 分类放入"工作流，并能够舒适地浏览和检索已有内容。

### 范围

1. 扫描用户指定的受管理根目录，通过增量扫描维护目录缓存。
2. 自动识别所有压缩包文件为内容单元候选，文件夹由用户手动标记。
3. 以基础分类目录树（一级或二级）展示资产库结构。
4. 提供双模式 UI：浏览模式（文件列表 + 内容单元标记 + 标签筛选 + 大图预览）和整理模式（暂存区操作 + 装配面板 + 快速插入）。
5. 支持用户标记任意目录为暂存区，在暂存区内创建 Mod 组（自动提取名称 → 建文件夹 → 移入文件）。
6. 支持从暂存区拖拽或一键插入到分类目录。
7. 为内容单元设置中文别名、标签、来源 URL、备注和封面。
8. 预置 Skyrim 标签库，支持自定义标签，支持标签分类筛选。
9. 全局搜索（标题 + 标签 + 备注）。
10. 基础文件操作：重命名（同步更新路径）、删除（移至回收站）、复制路径。
11. 操作历史与撤销。
12. 导出待整理内容单元列表为 JSON，提供给外部网页 AI 分类建议。
13. 导入 AI 建议的 JSON，在本地审阅并确认后应用。
14. 缩略图缓存与预览图展示。

### 非目标

第一版不包含以下功能：

- 云同步、账号、多人协作或远程数据库。
- 替代 MO2 的安装、排序、启用、冲突管理或游戏启动功能。
- 替代 Everything 的全盘文件搜索。
- 不经用户确认的自动文件移动、删除、覆盖或重命名。
- 从 Nexus Mods 自动批量抓取预览图。
- 解析压缩包内部图片或内容。
- 文件哈希去重、完整文件监听、MO2 深度联动。
- 本地 AI 推理或 AI 直接控制文件系统。
- 非 Mod 类型的内容单元元数据管理（教程、截图等暂通过文件系统直接浏览，不入库）。

## 4. 核心数据模型

### 4.1 ContentUnit（内容单元）

数据库记录的核心对象。每个内容单元对应一个真实路径（文件夹或单文件）。

```text
id: str                    # UUID
path: str                  # 真实路径（可为中文）
title: str | None          # 别名/中文显示名，为空则显示文件夹名
content_type: str          # "mod"（第一阶段仅此一种）
source_url: str | None     # 来源链接（如 N 网）
cover_path: str | None     # 封面图片路径（相对内容单元路径）
notes: str | None          # 备注
created_at: str            # ISO 8601 UTC
updated_at: str            # ISO 8601 UTC
```

### 4.2 TagCategory（标签分类）

标签的分组容器，用于 UI 筛选时的折叠/展开。

```text
id: str
name: str                  # 如"服装护甲"
color_hue: int             # 色相值，同一分类下标签颜色相近
```

### 4.3 Tag（标签）

```text
id: str
name: str                  # 如"重甲"
category_id: str           # 所属分类
```

一个标签只属于一个分类（一对一归属，不跨分类）。

### 4.4 ContentUnitTag（内容单元 ↔ 标签，多对多）

```text
content_unit_id: str
tag_id: str
```

### 4.5 OperationHistory（操作历史）

简化版操作记录，类似 PS 历史记录。

```text
id: str
operation_type: str        # move / delete / rename / new_folder
source_path: str
target_path: str | None
created_at: str
can_undo: bool             # 系统自动判断
```

### 4.6 ManagedRoot（受管理根目录）

保留现有设计。用于限定扫描范围。

### 4.7 FolderCache（目录树性能缓存）

简化版 folder_node，用于加速目录树显示。

```text
id: str
path: str
parent_id: str | None
last_scanned_mtime: float  # 上次扫描时的目录 mtime
created_at: str
```

### 4.8 ThumbnailCache（缩略图缓存）

保留现有设计。

### 4.9 已移除的旧实体

- ❌ ModItem → 由 ContentUnit 替代
- ❌ FileAsset → 不再以文件为粒度记录
- ❌ FileRole → 不再需要角色系统
- ❌ OperationLog（旧版四步状态机） → 由 OperationHistory 替代
- ❌ ConflictPolicy / OperationStatus / OperationType（旧版枚举）

## 5. 用户工作流

### 5.1 浏览模式（默认）

浏览模式的用途：浏览目录内容、查看和筛选内容单元、执行基础文件操作。

```
用户启动应用
    ↓
启动时自动扫描（增量扫描基于目录 mtime，跳过未变更目录）
    ↓
左侧目录树显示基础分类结构
    ↓
选中一个目录 → 中间显示该目录下的所有文件和文件夹
    ├─ 已被识别为内容单元的 → 显示 [内容单元] 标记和封面
    ├─ 未被识别为内容单元的 → 正常显示，无标记
    └─ 所有条目均可右键操作（复制路径、重命名、标记为内容单元等）
    ↓
在顶部标签栏筛选：
  ├─ 仅对已被标记为内容单元的项目生效
  ├─ 点击标签分类 → 展开标签 → 多选 → 列表实时筛选
  └─ 支持排序切换（名称/日期/类型/大小/状态 + 正序/倒序）
    ↓
单击内容单元 → 右栏立即显示元数据（详情面板交互方式）
双击文件夹（无论是否内容单元）→ 进入该目录（等价于目录树切换）
双击文件类型内容单元（压缩包）→ 显示元数据编辑面板
双击非内容单元文件 → 不响应
```

**关键设计**：
- 内容单元不是"可见性门槛"。文件系统里有什么，中间列表就显示什么。
  内容单元只是在文件基础上增加了**标记**和**元数据**，不影响文件本身是否可见。
- 元数据作为"详情面板"，单击查看更符合资源管理器/IDE/DAM 软件的交互习惯。
  文件夹的元数据通过单击查看；双击文件夹优先进入目录（而非显示元数据）。
- 扫描/创建 Mod 组等刷新操作后，目录树尽量保持原有展开状态与选中节点，
  避免用户每次都需要重新一级级展开。

### 5.2 整理模式

整理模式的用途：从暂存区整理零散文件、创建 Mod 组、分类放入、重命名、删除。

```
切换到整理模式
    ↓
中间区固定显示当前暂存区的内容（不再跟随树节点切换）
    ↓
目录树变为"目标选择器"：点击树节点只选中目标路径，不影响中间显示
    ↓
【创建 Mod 组】
  ├─ 在暂存区文件列表中选中一个本体文件（压缩包/文件夹）
  ├─ 右键 → "创建 Mod 组"
  │   ├─ 自动从文件名提取主要名称（剔除版本号、无用后缀等）
  │   ├─ 下拉框供选择：纯 Mod 名 / 完整原名
  │   └─ 确认后以所选名字创建真实文件夹，源文件移入
  │
  ├─ 中间区下方分割出"装配面板"，显示该文件夹的内容
  ├─ 右键暂存区文件 → "加入装配" 移入附加文件（汉化、预览图等）
  ├─ 预览图可通过装配面板右键手动重命名为 Mod 组同名
  └─ 装配完成
    ↓
【放入分类目录】
  ├─ 在目录树中选中目标分类（如 Armor/）
  ├─ 点击「快速插入」按钮 → 文件夹整体移至目标路径
  └─ 或直接拖拽文件夹到目录树的目标节点
    ↓
【直接操作】
  ├─ 拖拽文件到目录树节点 ← 从暂存区直接拖
  ├─ 树内拖拽重新分类
  ├─ 右键重命名（真实文件夹改名 + 同步更新 ContentUnit.path）
  ├─ 右键删除（移至回收站）
  └─ 右键复制路径
```

### 5.3 设置暂存区

```text
在目录树中右键任意目录 → "标记为暂存区" / "取消暂存区标记"
可以同时存在多个暂存区
暂存区在目录树中有视觉标记（如 [S] 前缀）
```

### 5.4 扫描

```text
启动时自动增量扫描：
  - 遍历每个已知目录，对比当前 mtime 与上次扫描记录
  - mtime 未变 → 跳过
  - mtime 变化 → 重新扫描该目录及子目录

内容单元识别规则：
  所有压缩包文件（.7z/.zip/.rar 等）均自动标记为内容单元。
  文件夹本身不自动标记，用户可手动标记。
  手动标记文件夹为内容单元时，其内部的所有内容单元标记自动取消。

第一阶段仅识别 Mod 类型内容单元（含压缩包）。
教程、截图等类型的目录不作识别，直接通过文件系统浏览。

用户可随时手动触发全量重新扫描。
不做实时文件系统监听（避免 CPU 负载影响游戏）。
```

### 5.5 AI JSON 交换（第一版后期）

```text
[优先级靠后，先完成手动整理体验]

导出：
  - 选中未整理的内容单元 → 导出 JSON（含路径、文件名、大小、当前标签）
  - 可选导出已有分类目录结构

导入：
  - 导入 AI 返回的建议 JSON
  - 审阅差异（建议中文名、建议分类、建议标签）
  - 逐项或批量勾选确认
  - AI 只生成建议，不直接操作文件系统
```

## 6. 文件操作安全规则

### 6.1 移动

| 规则 | 策略 |
|------|------|
| 节点间移动 | 弹出确认提示，防止误操作 |
| 重名冲突 | 当前阶段（阶段 3）：直接抛 ConflictError 拒绝，弹窗提示用户手动处理；覆盖 / 跳过 / 重命名选项留待阶段 5 通用冲突处理统一实现（AGENTS 规则 2：不覆盖） |
| 跨盘移动 | 提示"跨盘移动暂不支持"并拒绝（当前阶段跨盘场景较少，且 Windows 跨盘 move 会退化为 copy+delete，风险较高） |
| 文件夹移到自身或子目录 | 阻止并提示（此操作不合理） |

### 6.2 删除

- 统一移至 Windows 回收站。
- 操作历史记录删除操作，实际撤销能力由回收站提供。

### 6.3 重命名

- 在应用内重命名：同时修改真实文件夹名和 ContentUnit.path。
- 在应用外重命名：下次扫描检测旧路径丢失，需手动重新关联。

### 6.4 撤销

- 所有操作记录到 operation_history 表。
- 撤销时检查文件当前状态是否安全，不安全时阻止。

## 7. UI 结构

### 7.1 整体布局

```text
┌─── 顶部：标题 + 置顶按钮 + 搜索框 + [浏览|整理] 模式切换 ─┐
├──────────┬──────────────────────────┬─────────────────────┤
│          │                          │                     │
│  目录树   │    主内容区              │  元数据编辑面板     │
│  （左侧   │    （模式决定内容）       │                     │
│   固定）  │                          │  标题（别名）       │
│          │  浏览模式：               │  标签选择           │
│  AssetLib │  ┌─ 大图卡片/列表      │  来源 URL           │
│  ├─ Mods  │  ├─ 标签筛选栏         │  备注               │
│  │├Armor  │  └─ 排序切换           │  备注               │
│  │├Magic  │                          │  封面预览           │
│  ├─ Staging│  整理模式：               │  [保存]             │
│  │         │  ┌─ 暂存区文件列表       │                     │
│  │         │  ├─ 装配面板             │                     │
│  │         │  └─ [快速插入] 按钮      │                     │
└──────────┴──────────────────────────┴─────────────────────┘
```

### 7.2 浏览模式

- 点击目录树节点 → 中间显示该目录下的**所有文件和文件夹**
- 已被识别为内容单元的项目 → 显示内容单元标记（图标、封面）
- 未被识别为内容单元的项目 → 正常显示为普通文件/文件夹
- 右键菜单对所有项目可用（复制路径、重命名、删除）
- 未标记的文件/文件夹可右键 → "标记为内容单元"
- 已标记的内容单元可右键 → "取消标记"
- 顶部标签筛选栏：先选标签分类 → 展开标签 → 多选（仅过滤已标记的内容单元）
- 排序：名称 / 修改日期 / 类型 / 大小，正序/倒序切换
- **内容单元不是可见性门槛**。所有文件系统内容都能在应用中看到和操作。
- **交互方式（2026-07-16 调整）**：
  - 单击选中内容单元 → 右栏立即显示元数据（详情面板交互方式）
  - 双击文件夹（无论是否内容单元）→ 进入该目录（等价于目录树切换）
  - 双击文件类型内容单元（压缩包）→ 显示元数据编辑面板
  - 双击非内容单元文件 → 不响应
- **展开状态保持**：扫描/创建 Mod 组等刷新操作后，目录树尽量保持原有展开状态与选中节点。

### 7.3 整理模式

- 切换到整理模式后，中间区固定显示当前暂存区内容，不随目录树切换
- 目录树变为目标选择器，点击仅选中目标路径
- 暂存区文件列表：显示暂存区内所有零散文件
- 创建 Mod 组后分割出装配面板
- **交互方式（2026-07-17 调整）**：
  - 单击 Mod 组文件夹 → 仅选中并显示元数据，**不切换**装配面板绑定（避免误触）
  - 双击 Mod 组文件夹 → 将装配面板绑定到该 Mod 组并刷新内容
  - 双击普通文件 / 普通文件夹 → 不响应
- 「快速插入」按钮：当前装配面板绑定的 Mod 组 → 目录树中选中的目标路径
  （2026-07-17 阶段 3 Task 5 实现）
  - **可用条件**：整理模式 + 装配面板已绑定 Mod 组 + 目录树选中目标目录 + 目标与源不同 + 目标不是源子目录
  - **交互**：点击按钮 → 弹出确认对话框（显示源路径 → 目标路径）→ 用户确认后执行移动
  - **底层**：调用 QuickInsertService.quick_insert，移动后更新 ContentUnit.path、清理旧 folder_cache、写入 operation_history
  - **UI 刷新**：成功后解绑装配面板 + 刷新目录树 + 刷新暂存区列表 + 状态栏提示
  - **取消拖拽方案**：原 spec 中"拖拽到目录树节点"和"目录树内拖拽重新分类"取消（与 Task 4 拖拽取消决策一致），仅保留「快速插入」按钮入口

### 7.4 装配面板

- 创建 Mod 组后自动出现（整理模式下）
- 显示当前选中 Mod 组文件夹的内容
- 通过暂存区文件列表右键菜单「加入装配」移入附加文件（汉化、预览图等）
  （2026-07-17 调整：取消原拖拽方案，改用右键菜单触发，底层业务逻辑不变）
- 支持移除已加入的文件（移回暂存区根目录，不保留原子目录结构）
- **不自动重命名图片**：自动整理阶段不修改任何文件名，避免破坏用户已有命名
- **手动重命名预览图**：在装配面板中右键图片 → "重命名为与 Mod 组同名"，
  执行后图片重命名为 `{Mod组名}.{扩展名}`；多张图片时采用
  `{Mod组名}_2.{扩展名}`、`{Mod组名}_3.{扩展名}`……后缀；
  命名冲突走现有 `ConflictError` 流程（弹窗提示冲突，不覆盖，AGENTS 规则 2）
- 整理模式下切换不同 Mod 组时，装配面板同步刷新内容
- 浏览模式不显示装配面板

### 7.5 右键菜单

| 场景 | 菜单项 |
|------|--------|
| 暂存区中的文件 | 创建 Mod 组、加入装配（仅整理模式 + 装配面板已绑定 Mod 组）、复制路径、删除（移至回收站） |
| 目录树中的内容单元 | 重命名、复制路径、删除（移至回收站） |
| 目录树中的目录 | 标记/取消暂存区、复制路径、重命名 |
| 内容单元列表 | 编辑元数据、批量打标签、复制路径、在浏览器打开（有来源链接时） |
| 装配面板中的图片 | 重命名为与 Mod 组同名、移除（移回暂存区）、复制路径 |
| 装配面板中的非图片文件 | 移除（移回暂存区）、复制路径 |

### 7.6 置顶按钮

窗口右上角或工具栏有一个"置顶"切换按钮，全屏操作时窗口不会被其他程序遮挡。

## 9. 预览图

- 内容单元可从其目录内选择一张图片作为封面。
- **封面自动候选**：默认自动选取内容单元目录内第一张图片作为封面候选，用户可确认或手动更换。
- 支持常见图片格式：JPG、JPEG、PNG、WEBP、GIF、BMP、TIF、TIFF、ICO。
- 应用生成的缩略图缓存保存到应用数据目录，不修改原始图片。
- 缩略图缓存有效性基于 `content_unit_id + source_size + source_modified_at`。
- 浏览模式下大图卡片展示封面；列表模式下小图标展示。
- 无封面的内容单元退化为文件列表样式。

## 10. 标签系统

### 10.1 标签分类结构

```text
TagCategory（标签分类）        Tag（标签）
服装护甲（蓝色系）             重甲, 轻甲, 法袍, 现代, 幻想, 裸露, 内衣, 泳装, 饰品, 杂项, 合集
武器（橙色系）                 单手剑, 双手剑, 弓, 法杖
作者（绿色系）                 （用户自行添加）
来源                           N网, L网, 韩网, 群友分享, 私货
状态                          已测试, 已汉化, 待测试
```

### 10.2 创建方式

- **预制标签库**：以 `default_tags.json` 文件形式随应用发布，首次启动或标签表为空时自动导入数据库。
- 用户可自定义新标签（打字输入 + 自动补全已有标签）。
- 支持将当前标签库导出为 JSON 文件（备份或批量修改后重新导入）。
- 可选 AI 扫描辅助建议。

### 10.3 UI 交互

- 筛选：先点分类 → 展开标签 → 多选高亮标签 → 筛选内容单元列表
- 打标签：打字自动补全 + 最近常用置顶
- 批量操作：选中多个内容单元 → 一次性添加/移除标签
- **标签管理对话框**：入口在浏览模式顶部或元数据面板旁，提供：
  - 标签分类和标签的增删改
  - 「导入标签」按钮（QFileDialog 选 JSON → 导入数据库）
  - 「导出标签」按钮（QFileDialog 选保存位置 → 导出为 JSON）
